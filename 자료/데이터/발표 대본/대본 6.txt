(페이지 넘김)

저희가 선정한 주제는 졸업요건 조회 사이트입니다.

(페이지 넘김)

먼저 DB 테이블 입니다.
기존의 DB 테이블은 학과, 학번, 전공필수, 교양, 게시판 테이블이었지만
회의 결과, 학번 테이블은 삭제하고 각 학번의 맞는 교양 졸업요건은 코드로 구현하는 방향이 더 좋을 것 같다고 생각해서 다음 사진과 같이 결정했습니다.

(페이지 넘김)

다음은 UI 입니다.
현재 메인페이지, 인기교양추천 페이지는 어느 정도 모습을 갖춘 상태입니다.
졸업요건 확인 페이지는 업로드 버튼만 만들어져 있고 결과 페이지는 아직 미완성입니다.

(페이지 넘김)

저번 주 발표에서, 졸업요건 결과조회 기능에서 어떻게 결과조회를 하는지 구체적인 내용이 있었으면 좋겠다는 피드백이 있었습니다.
졸업요건 결과조회는 졸업요건 검사 결과에 따라 제공되는데, 이 졸업요건 검사는 학사지원팀으로부터 제공받은 졸업불가사유 6가지를 바탕으로 이루어집니다.

학사지원팀으로부터 받은 졸업불가사유 6가지는
교양학점 미달, 전공 최소학점 미달, 졸업학점 미달, 교필 미이수, 졸논 미이수, 전필 미이수입니다.

(페이지 넘김)

졸업요건 검사 순서에 대해 간단하게 설명드리겠습니다.
먼저 pdf 파일이 업로드되면 해당 파일이 학업성적확인서 pdf 파일인지 확인합니다.

다음으로 pdf 파일에서 학번, 학과, 총 취득학점 등 검사에 기본적으로 필요한 정보를 추출합니다.

이어서 학점 검사를 진행합니다.
총 취득학점이 졸업학점을 넘는지 검사하고 검사 결과가 졸업학점을 넘지 않는다면 몇 학점 남았는지 제공합니다.

(페이지 넘김)

졸업학점 다음으로는 교양학점과 전공 최소이수학점을 검사합니다.
교양학점은 해당 학번의 교양 이수학점 범위내에 있는지 검사하고, 전공 최소이수학점은 해당 학과의 전공 최소이수학점을 넘는지 검사하여
만족하지 못한 경우 몇 학점을 더 들어야 하는지 제공합니다.

(페이지 넘김)

다음으로 전공필수 과목을 검사합니다.
해당 학과의 전공필수 목록을 데이터베이스로부터 가져와서 전부 들었는지 검사하고, 전부 듣지 않았다면 들어야 할 과목들을 제공합니다.

교양필수도 전공필수와 같은 방식으로 검사합니다.

(페이지 넘김)

비교과는 학번마다 기준이 달라 해당 학번의 기준을 따릅니다.
해당 학번 기준을 넘었으면 통과, 그렇지 않으면 부족한 이수학기 또는 마일리지 점수를 제공합니다.

마지막으로 19학번부터 생긴 교양 조건에 대한 검사입니다.
19학번은 교양을 인재상별로 5학점 이상 들어야 하고, 20학번 이후는 교양을 핵심역량별로 1과목 이상 들어야 합니다.

저희는 모든 교양의 인재상과 핵심역량도 정리하여 교양 테이블에 데이터를 넣었습니다.
교양 테이블에서 가져온 정보와 pdf에서 읽어온 과목명을 비교하여 해당 학번의 기준을 넘었다면 통과, 넘지 않았다면 인재상 또는 핵심역량별 부족한 부분을 제공할 예정입니다.

(페이지 넘김)


다음은 비기능 요구사항입니다.

성능은 학업 성적 확인서를 업로드한 다음 결과가 나오기까지 5초를 넘기면 안 되고 게시판 업로드는 3초를 넘기면 안 됩니다.

신뢰성은 학업성적 확인서를 올렸을 때 실패는 5% 미만이어야 합니다.

사용 용이성은 업로드 버튼을 누르고 올리면 바로 결과 창이 나옵니다.


(페이지 넘김)


유지 보수성은 새로 추가된 과목들을 추가할 수 있도록 해야 합니다.

효율성은 PDF 업로드 시 학업성적 확인서 하나만 업로드 해야 합니다.


(페이지 넘김)

앞서 말씀드렸던 졸업요건 검사 조건 순서를 적용하여 일부 기능을 구현하였습니다.
먼저 PDFBOX를 사용하여 업로드된 PDF 파일의 내용을 읽어옵니다.

(페이지 넘김)

JDBC를 이용하여 데이터베이스에서 필요한 데이터를 가져옵니다.
보고 계시는 사진은 JDBC를 이용해 테이블에서 데이터를 가져올 때 필요한 쿼리문들입니다.

(페이지 넘김)

학업성적확인서를 업로드 시 한신대학교 학업성적확인서가 맞는지 검사하는 코드입니다.

(페이지 넘김)

다음은 학생의 학번, 학과, 총 취득학점, 교양, 전공 이수학점, 비교과 이수 학기, 마일리지와 같은 기본정보를 pdf 파일의 내용으로부터 추출하는 코드입니다.

(페이지 넘김)

다음은 졸업 학점을 검사하는 코드입니다.

첫번째로 졸업학점을 검사합니다.
pdf에서 추출해온 총 취득학점이 130학점이 넘는지 검사하여 넘지 않으면 부족한 학점을 제공합니다.

두번째로 교양 학점을 검사합니다.
학번마다 교양 학점 기준이 다르다는 점을 고려하여 구현했습니다.

세번째로 전공 최소이수학점을 검사합니다.
데이터베이스에서 해당 학과의 전공 최소이수학점을 가져와 사용자의 전공학점과 비교하는 방식입니다.
이 역시 학점이 부족하면 부족한 학점을 제공합니다.

(페이지 넘김)

마지막으로 현재까지 구현된 테스트 결과입니다.
pdf를 읽어와 한신대학교 학업성적확인서인지를 확인하고, 학생 기본 정보를 출력하고, 부족한 졸업요건을 간단하게 출력해보았습니다.
